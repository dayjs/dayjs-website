Roadmap

Development roadmap for the dayjs community.

#Why started this

The original idea of the roadmap can be founded in https://github.com/iamkun/dayjs/issues/627.

We want to make dayjs to cover more languages. Like day.rs (In Rust), day.clj (In Clojure) and so on.

At the same time, we also want to migrate the existing documentation to a new website.

There will be a lot of work to be done so we started this roadmap.

The rest of the content of this roadmap is divided into 3 parts:

  • Languages Coverage
  • Document Migration Progress
  • How To Contribute

If you want to know what languages dayjs can also be used in, click Languages Coverage.

If you want to know the progress of newest documentation, go to Document Migration Progress.

If you want to contribute to dayjs community, like add new language support, fixes documentation problems, go to How To Contribute.

We are going to use https://v2.docusaurus.io/ to generate the new documentation.

Current progress: 50%

  • 2019-07-17: We have already migrated the en docs of dayjs.

#How To Contribute

#Add new language support

Open an issue and describe your plan, we will reply you as soon as possible.

If you want to add new language support, we suggest you to views the issues first. Because maybe someone already starts to work on.

#Fixes documentation problems

Go to the website repo, open an issue or pull a request to help us.
